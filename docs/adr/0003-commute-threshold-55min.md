# 通勤阈值由 ≤40 分放宽到门到门 ≤55 分

PRD 初版把命中条件 #1 定为「到勝どき换乘 ≤ 40 分的神奈川団地」。在 issue 02 用乗換案内(Yahoo!路線情報 / ジョルダン / 駅探 / NAVITIME)实测 46 个候选站到勝どき的所要時間后发现:**勝どき駅只有都営大江戸線一条线**,神奈川方向几乎都要换 1~3 次车,门到门 ≤40 分的団地**仅剩 1 个**(小杉御殿,武蔵小杉 徒歩9 + 鉄道 29 = 38 分)。连原本被预期为「明显近」的 川崎(41)/横浜(42)/日吉(47)/鶴見(~50)/元住吉(~40–50),加上団地到站徒歩后也都 >40 分。

≤40 的 watchlist 只有 1 个団地,基本失去监控意义。经与用户确认,把阈值放宽到**门到门 ≤ 55 分**,筛出 **18 个団地**(武蔵小杉 / 川崎 / 横浜 / 鶴見 / 元住吉 / 港北一带)。完整实测数据见 `.scratch/ur-vacancy-alert/data/commute_to_katsudoki.md`。

取舍依据:watchlist 的**漏报(砍掉其实够近的団地)代价高**(会静默错过房源),**误报(多收一个稍远団地的通知)代价低**(本人看一眼忽略即可),故阈值附近偏向纳入。

## Consequences

- `watchlist.json` 含 18 个団地,每項带 `commute_min`(门到门估算分钟)便于追溯与重排。
- 通勤数值中,实测站(V)误差小,推算站(E,如港北 NT 的グリーンライン各站)误差 ±5~8 分;56~57 分的边界団地(川崎旭町ハイツ / アーバンハイツみぞのくち / シティコート元住吉 等)被排除在外,如需更宽的网可手动加入。
- 电车时刻表少变,该阈值与 watchlist 偶尔重算即可(重跑 `commute_to_katsudoki.md` 的调研)。
- 命中条件 #1、README、邮件正文条件行均已同步为 `≤55分(门到门)`。
