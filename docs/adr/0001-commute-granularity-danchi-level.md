# 通勤精度定为団地级,棟级推迟

UR 的数据源只公布**団地级**的位置(代表地址、最寄り駅 + 徒歩分、单一坐标),不提供每一棟的坐标;同団地内不同栋号的地址做地理编码通常会落到同一点,无法可靠区分。因此本产品的**通勤时间一律基于団地坐标计算**,栋号仅作为部屋的展示属性保留,不参与通勤筛选。

棟级通勤需要自建一套「栋号 ↔ 建筑轮廓坐标」的数据集(如 OSM 建筑轮廓 + 人工核对),成本高、维护重,推迟到 v2 作为独立子项目评估。

## Consequences

- 筛选、排序、通知中的「通勤时间」都以団地为单位;同団地下所有部屋共享同一通勤时间。
- 大団地(横跨数百米)存在团内步行差异,v1 不在通勤数值里体现 —— 必要时可在 UI 上提示"団地较大,各栋到站时间略有差异"。
