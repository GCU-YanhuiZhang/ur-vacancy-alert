# 自用定时脚本,不做网站

最初的设想是一个面向多用户、可在浏览器里调筛选条件的网站。但真实需求是**单人自用**,且筛选条件全部固定(神奈川県 / 到勝どき换乘 ≤ 40 分 / + 家賃间取り等)。在这种前提下,网站的核心卖点(让用户自助调条件、多用户、账号)约 80% 用不上。

**决定:做一个定时运行的个人脚本**,流水线为「爬取神奈川 UR 空房 → 按固定条件筛 → 与上次结果对比、找出新出现的部屋 → 发邮件给本人」。**不做** Web 前端、账号系统、用户数据库。

## Consequences

- 「筛选条件」从运行时输入退化为代码/配置里的常量。
- 把工程重心从"全栈网站"转移到这个产品真正难的地方:爬得稳、判得准、邮件不漏不重。
- 若将来要开放给他人使用,需重新评估架构(加回账号、多租户筛选、前端)—— 届时本 ADR 标记为 superseded。
