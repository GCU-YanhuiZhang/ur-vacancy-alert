# UR 空房通知 (UR Vacancy Alert)

一个**自用**的定时脚本:自动抓取 UR 賃貸住宅空房信息、按一组**固定**条件(神奈川県 / 到勝どき门到门 ≤ 55 分等)匹配、并在出现新空房时通过邮件通知本人。非网站、单租户(见 ADR-0002)。

## Language

### 房源结构 (Housing hierarchy)

**団地 (Property)**:
一整个 UR 住宅区或大规模公寓,有名字与所在区域,下辖多个棟。UR 自己的 API 里称「物件 / bukken」。
_Avoid_: 小区、楼盘、mansion、物业

**棟 (Building)**:
団地下的一栋具体楼,有自己的物理位置。
_Avoid_: block、楼(正式词统一用「棟」)

**部屋 (Unit)**:
棟里的一间具体住戸,是空房、家賃、間取り的最小单位,各有自己的家賃、間取り、楼层、面积、朝向。UR API 里称「住戸 / room」。
_Avoid_: 户、住宅(「房间」口语可)

### 核心事件

**空房事件 (Vacancy)**:
某个具体「部屋」从「不在 UR 可申请列表」变为「在可申请列表」的那一刻。这是通知与去重的最小触发单位 —— 触发条件是部屋级的,**不是**「某団地有没有空房」的布尔值。
_Avoid_: 空室(UR 原词;中文统一用「空房事件」指这个事件)、放盘、上架

### 通勤 (Commute)

**目标站 (Target station)**:
希望快速通勤到达的车站。自用前提下**固定为勝どき**(将来若多用户化才会变成用户输入)。
_Avoid_: 目的地、终点站

**通勤时间 (Commute time)**:
从団地到「目标站」乘坐公共交通(含换乘)所需的分钟数。本产品所说的「通勤距离」一律指这个时间,**不**指直线距离或步行距离。粒度为団地级(见 ADR-0001)。
_Avoid_: 通勤距离(易歧义)、车程、路程

**最寄り駅 (Nearest station)**:
距离団地最近、可步行到达的车站,由 UR 数据自带(含徒歩分),作为计算通勤时间的起点。
_Avoid_: 最近站(口语可)、起点站

### 匹配 (Matching)

**通勤候选団地 (Watchlist)**:
到勝どき门到门 ≤ 55 分的神奈川団地集合(阈值由 ≤40 放宽,见 ADR-0003)。脚本只监控这个集合内的団地的空房。
_Avoid_: 白名单、监控列表

**命中 (Match)**:
一个空房事件,其所属団地在通勤候选団地内,且同时满足全部固定筛选条件(家賃、間取り、面积)。只有命中才会触发邮件。
_Avoid_: 合格、符合(「匹配」作动词可)
